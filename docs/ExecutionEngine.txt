The Execution Engine is the module that actually places trades on exchanges after the Strategy Engine decides what to do. It is responsible for order placement, position management, SL/TP updates, and trade monitoring. Below is a complete detailed architecture and implementation guide.

⸻

1️⃣ Purpose of the Execution Engine

The execution engine converts strategy decisions into real orders on the exchange.

Flow:

Strategy Engine
      ↓
Execution Engine
      ↓
Exchange API
      ↓
Trade Executed

Responsibilities:
	•	Place market or limit orders
	•	Place stop loss and take profit
	•	Update SL to break-even
	•	Track open positions
	•	Handle exchange errors
	•	Log trades in database

3️⃣ Exchange Client

This module handles communication with exchanges.

Use CCXT because it supports many exchanges:
	•	Binance
	•	Bybit
	•	OKX
	•	KuCoin

Install:

pip install ccxt

Example client:

import ccxt

class ExchangeClient:

    def __init__(self, api_key, secret):

        self.exchange = ccxt.binance({
            "apiKey": api_key,
            "secret": secret,
            "enableRateLimit": True
        })

    def get_price(self, symbol):

        ticker = self.exchange.fetch_ticker(symbol)

        return ticker["last"]

4️⃣ Order Manager

Handles placing orders.

Example:

class OrderManager:

    def __init__(self, exchange):
        self.exchange = exchange

    def market_buy(self, symbol, amount):

        return self.exchange.create_market_buy_order(
            symbol,
            amount
        )

    def market_sell(self, symbol, amount):

        return self.exchange.create_market_sell_order(
            symbol,
            amount
        )

6️⃣ Stop Loss Order

Example:

def place_stop_loss(symbol, side, amount, stop_price):

    order_side = "sell" if side == "buy" else "buy"

    return exchange.create_order(
        symbol=symbol,
        type="stop",
        side=order_side,
        amount=amount,
        params={"stopPrice": stop_price}
    )

8️⃣ Execution Engine Core

This module connects strategy decisions with order manager.

Example:

class ExecutionEngine:

    def __init__(self, exchange_client):

        self.exchange = exchange_client
        self.order_manager = OrderManager(exchange_client.exchange)


⸻

Open Trade

def open_trade(self, symbol, side, entry, sl, tp, amount):

    order = self.order_manager.market_buy(symbol, amount) \
        if side == "buy" \
        else self.order_manager.market_sell(symbol, amount)

    self.order_manager.place_stop_loss(symbol, side, amount, sl)

    self.order_manager.place_take_profit(symbol, side, amount, tp)

    return order

9️⃣ Break-Even Update

When strategy triggers BE:

SL = entry

Update stop loss.

Example:

def update_stop_loss(symbol, side, amount, new_sl):

    order_side = "sell" if side == "buy" else "buy"

    exchange.create_order(
        symbol=symbol,
        type="stop",
        side=order_side,
        amount=amount,
        params={"stopPrice": new_sl}
    )

12️⃣ Error Handling

Exchanges often fail due to:
	•	rate limits
	•	network errors
	•	rejected orders

Example retry logic:

def safe_order(func, *args):

    try:
        return func(*args)

    except Exception as e:

        print("Order failed, retrying")

        return func(*args)

15️⃣ Handling Reversal Trades

If SL hits before BE:

close trade
open opposite trade

Example:

def reverse_trade(position):

    side = "sell" if position["side"] == "buy" else "buy"

    open_trade(symbol, side)
